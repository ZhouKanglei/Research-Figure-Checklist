# Research Figure Checklist

科研绘图不是简单地“把数据画出来”，而是将实验结果转化为论文论证的一部分。

本仓库整理了笔者在论文写作、科研绘图与学生指导中的实践经验，涵盖绘图原则、检查清单、配色方案、代码模板与案例分析，帮助科研新手绘制更清晰、规范、统一且可复现的论文插图，更有效地展示实验结果、方法优势与论文创新点。

示例主要来自计算机科学领域，尤其是机器学习与计算机视觉方向；但其中关于图表设计、配色、排版和论文表达的多数原则，也适用于其他学科的科研绘图。

---

## 项目目标

本仓库希望帮助科研新手解决论文绘图中的常见问题：

- 不知道一张图应该突出什么结论；
- 图表只是展示数据，没有服务论文叙事；
- 配色随意，不同图之间风格不统一；
- 字号、线宽、图例在论文中缩小后不可读；
- 只保存位图，没有保存适合投稿的矢量图；
- caption 只描述图中内容，没有总结核心发现；
- 绘图代码不可复现，后续修改困难。

本项目的核心目标是帮助大家从：

> “把图画出来”

逐步过渡到：

> “用图清晰、可信、有说服力地表达论文贡献”。

## 主要模块

### 1\. `checklists/`：科研绘图检查清单

用于论文投稿、组会汇报或论文修改前的自查。

计划包括：

-   通用科研绘图 checklist；
    
-   投稿前图表 checklist；
    
-   配色 checklist；
    
-   Matplotlib 绘图 checklist；
    
-   LaTeX 插图 checklist；
    
-   Rebuttal / Revision 图表 checklist。
    

---

### 2\. `docs/`：科研绘图知识

系统整理科研绘图中的基础知识与经验，包括：

-   什么是好的科研图；
    
-   如何选择合适的图类型；
    
-   柱状图、折线图、热力图、箱线图等常见图的使用场景；
    
-   如何设计论文图配色；
    
-   如何统一字体、字号、线宽和布局；
    
-   如何写高质量 figure caption；
    
-   常见绘图问题与修改建议。
    

---

### 3\. `templates/`：可复用绘图模板

提供基于 Python / Matplotlib 的论文绘图模板。

计划包括：

-   bar plot；
    
-   line plot；
    
-   heatmap；
    
-   scatter plot；
    
-   box plot；
    
-   radar plot；
    
-   multi-panel figure。
    

模板不仅关注“画出来”，也关注：

-   代码结构是否清晰；
    
-   图表风格是否统一；
    
-   是否便于替换数据；
    
-   是否适合论文投稿；
    
-   是否可以复现和复用。
    

---

### 4\. `palettes/`：论文配色方案

整理常见科研绘图配色方案，例如：

-   Nature-style palette；
    
-   Science-style palette；
    
-   soft academic palette；
    
-   colorblind-friendly palette；
    
-   baseline-vs-ours palette；
    
-   sequential / diverging colormap。
    

基本原则是：

> baseline 低调，ours 突出；辅助信息克制，核心结论醒目。

---

### 5\. `cases/`：案例分析

通过具体案例展示如何从实验数据得到论文图表。

每个案例建议包含：

```
case_name/  
├── README.md      # 案例说明  
├── data.csv       # 示例数据  
├── plot.py        # 绘图代码  
├── figure.pdf     # 论文用矢量图  
└── figure.png     # 预览图
```

案例内容包括：

-   这张图想说明什么；
    
-   为什么选择这种图类型；
    
-   数据如何组织；
    
-   配色如何设计；
    
-   代码中有哪些关键设置；
    
-   caption 应该如何写；
    
-   常见错误有哪些。

## 推荐使用流程

绘制科研论文图表时，可以按照以下流程进行：

1.  明确图的核心结论；
    
2.  判断是否真的需要画图，还是表格更合适；
    
3.  根据表达目标选择合适的图类型；
    
4.  整理干净、可复现的数据文件；
    
5.  选择统一且克制的配色；
    
6.  编写可复用的绘图代码；
    
7.  导出 PDF / SVG / EPS 等矢量格式；
    
8.  同时保存高分辨率 PNG 用于预览和汇报；
    
9.  检查图缩放到论文单栏或双栏后的可读性；
    
10.  编写能够概括核心发现的 caption；
    
11.  在正文中解释这张图如何支持论文结论。

## 通用科研绘图 Checklist

### 图的目的

-    这张图是否服务于一个明确结论？
    
-    读者能否在几秒内看出主要信息？
    
-    这张图是否支持论文的某个核心观点？
    
-    这张图是否比表格更适合表达当前结果？
    
-    正文是否充分解释了这张图？
    

### 数据表达

-    是否完整展示了关键对比？
    
-    是否避免选择性展示结果？
    
-    坐标轴范围是否合理？
    
-    是否避免通过截断坐标轴夸大差异？
    
-    多个子图之间的坐标范围是否统一或有明确说明？
    

### 视觉设计

-    字体、字号、线宽是否统一？
    
-    图例是否清晰且不遮挡数据？
    
-    坐标轴标签是否简洁明确？
    
-    子图编号如 `(a), (b), (c)` 是否统一？
    
-    是否去除了不必要的边框、网格和装饰元素？
    
-    图在缩小到论文宽度后是否仍然可读？
    

### 配色

-    颜色是否具有明确含义？
    
-    baseline 和 proposed method 是否区分清楚？
    
-    是否避免使用过多颜色？
    
-    同一方法在不同图中是否使用一致颜色？
    
-    是否考虑色盲友好性？
    
-    热力图色条是否清晰标注？
    

### 投稿规范

-    是否保存为 PDF / SVG / EPS 等矢量格式？
    
-    是否同时保存高分辨率 PNG？
    
-    是否避免截图式插图？
    
-    字体是否能够正确嵌入？
    
-    LaTeX 插图后是否存在明显白边、模糊或字体过小问题？
    
-    图表文件命名是否清晰？
    

### Caption 与正文

-    caption 是否说明图展示的内容？
    
-    caption 是否总结核心发现？
    
-    caption 是否避免过于笼统，例如只写 “Comparison of different methods”？
    
-    正文是否引用并解释该图？
    
-    图、caption 和正文分析是否一致？
    
-    caption 是否能够帮助读者在不读正文的情况下理解图的主要结论？

## 基础 Matplotlib 设置

下面是一组常用的论文绘图初始设置：

```Python
import matplotlib.pyplot as plt  
  
plt.rcParams["font.family"] ="Arial"  
plt.rcParams["font.size"] =10  
plt.rcParams["axes.linewidth"] =0.8  
plt.rcParams["pdf.fonttype"] =42  
plt.rcParams["ps.fonttype"] =42
```


保存图片时，建议同时保存矢量图和高分辨率位图：

```Python

plt.savefig("figure.pdf", bbox_inches="tight")  
plt.savefig("figure.png", dpi=600, bbox_inches="tight", transparent=True)
```
```

其中：

-   `PDF / SVG / EPS` 更适合论文投稿；
    
-   `PNG` 更适合预览和汇报；
    
-   `bbox_inches="tight"` 可以减少多余白边；
    
-   `transparent=True` 方便放入论文或幻灯片。

## Caption 示例

较弱的 caption：

```
LaTeX

```
\caption{Comparison of different methods.}
```
```

更好的 caption：

```
LaTeX

```
\caption{  
Performance comparison between the baseline method and our approach under different evaluation settings.  
Our method consistently improves accuracy across all settings, with more pronounced gains under the challenging continual learning scenario.  
}
```
```

好的 caption 不应该只是说明“图里有什么”，还应该说明“图说明了什么”。
