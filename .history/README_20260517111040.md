# Research Figure Checklist

科研图表从来不只是“做得好看”。它需要让读者快速理解研究问题、方法路径、关键结果和结论边界。本项目采用“指南 + 清单 + 案例 + 模板”的结构，帮助作者在投稿、汇报、论文返修和组会展示前系统检查图表质量。

## 指南

### 1. 明确图表目标

每张图都应回答一个具体问题。开始绘图前先写下一句话：

> 这张图要让读者相信什么？

常见目标包括：

- 展示研究设计或实验流程
- 比较不同组别、模型或条件
- 说明变量之间的关系
- 呈现空间、时间或剂量响应趋势
- 总结机制、框架或假设

如果一张图同时承担太多目标，优先拆成多个 panel，或将部分信息移到补充材料。

### 2. 组织图表层级

推荐使用从整体到细节的阅读顺序：

1. 标题或图注先交代核心结论
2. Panel A 展示研究设计、数据来源或分析流程
3. 中间 panel 支撑主要发现
4. 最后 panel 汇总结论、机制或外部验证

多 panel 图应让读者即使只看图，也能理解基本逻辑。

### 3. 选择合适图形

图形类型应服务于数据结构，而不是视觉偏好。

| 数据目的 | 推荐图形 | 避免 |
| --- | --- | --- |
| 组间比较 | 点图、箱线图、小提琴图、估计区间图 | 只用柱状图隐藏分布 |
| 相关关系 | 散点图、回归线、残差图 | 用双 y 轴制造关系 |
| 时间变化 | 折线图、轨迹图、热图 | 过多重叠线条 |
| 组成比例 | 堆叠条形图、马赛克图 | 3D 饼图 |
| 空间分布 | 地图、组织切片、空间热图 | 无比例尺的图像 |
| 流程机制 | 流程图、机制图、因果图 | 装饰性箭头堆叠 |

### 4. 保证可读性

图表应在论文双栏、PPT 投影和打印版本中都能读清。

- 字号：正文图中建议不低于 7-8 pt，展示图中不低于 14-16 pt
- 线宽：避免过细；导出后检查缩放状态
- 颜色：优先使用色盲友好配色
- 标签：直接标注关键线条或组别，减少读者反复查图例
- 留白：保留足够边距，避免元素贴边

### 5. 保持科研诚信

图像和统计可视化都应可追溯、可复现。

- 不做误导性裁剪或非线性缩放
- 显微图、胶图、影像图保留比例尺和处理说明
- 所有统计图说明样本量、误差定义和检验方法
- 对图像增强、去噪、归一化等处理进行记录
- 原始数据、代码和导出参数尽量归档

## 清单

### 投稿前总清单

- [ ] 每张图都有明确的科学问题或核心结论
- [ ] Panel 顺序符合论文叙事逻辑
- [ ] 图注可以独立解释数据来源、统计方法和缩写
- [ ] 坐标轴包含变量名、单位和必要的变换说明
- [ ] 样本量 `n`、重复次数和统计检验方法已标注
- [ ] 误差线含义明确，例如 SD、SEM、95% CI 或 IQR
- [ ] 颜色、符号和线型在全文中保持一致
- [ ] 所有缩写首次出现时已解释
- [ ] 图中文字在目标版式下清晰可读
- [ ] 图片分辨率、格式和尺寸符合期刊要求
- [ ] 原始数据、绘图代码和最终导出文件已归档

### 数据图清单

- [ ] 图形类型与数据类型匹配
- [ ] 没有用柱状图替代必要的分布信息
- [ ] 坐标轴没有截断或误导性压缩
- [ ] 离群值处理有说明
- [ ] 组间比较使用统一尺度
- [ ] 多重比较校正方法已说明
- [ ] 图中标注的显著性与统计结果一致
- [ ] 颜色不作为唯一编码方式，必要时配合形状或线型

### 图像类清单

- [ ] 包含比例尺、方向或空间定位信息
- [ ] 对比度、亮度、伪彩和合成方式说明清楚
- [ ] 不同组别使用相同处理参数
- [ ] 裁剪区域具有代表性
- [ ] 原图和处理后图像可追溯
- [ ] 关键区域有标注，但不遮挡原始信号

### 机制图与流程图清单

- [ ] 箭头方向表达清楚
- [ ] 节点命名统一，避免同义词混用
- [ ] 流程从左到右或从上到下阅读
- [ ] 只保留支撑主结论的关键步骤
- [ ] 图例解释符号、颜色和线型含义

### 可访问性清单

- [ ] 使用色盲友好配色
- [ ] 灰度打印后仍能区分主要元素
- [ ] 关键结论不只依赖颜色表达
- [ ] 图注避免含糊描述，例如“明显增加”
- [ ] 文件名能反映图号、版本和用途

## 案例

### 案例 1：组间比较图

**问题**

使用柱状图展示 3 组实验结果，只显示均值和 SEM，读者无法判断数据分布、样本量和离群值。

**改进**

- 改为散点叠加箱线图或估计区间图
- 标注每组样本量
- 在图注中说明误差线含义和统计检验
- 使用统一 y 轴范围，并避免不必要的截断

**检查重点**

- 数据点是否可见
- 组间比较是否使用同一尺度
- 显著性标注是否与统计方法一致

### 案例 2：多 panel 论文主图

**问题**

Panel A-D 分别展示数据来源、模型性能、特征重要性和外部验证，但顺序混乱，图注没有解释缩写。

**改进**

- Panel A：研究设计和数据来源
- Panel B：主要模型性能
- Panel C：关键变量或机制解释
- Panel D：外部验证或敏感性分析
- 图注按 panel 顺序逐一说明数据、方法和缩写

**检查重点**

- 读者能否按 panel 顺序理解研究逻辑
- 每个 panel 是否只承担一个核心任务
- 图注是否能独立阅读

### 案例 3：显微图或影像图

**问题**

图像经过增强处理，但没有说明处理参数；比例尺缺失；不同组别亮度范围不一致。

**改进**

- 添加比例尺
- 统一所有组别的显示范围
- 在图注中说明图像处理步骤
- 保存原图、处理脚本和导出版本

**检查重点**

- 图像处理是否影响定量结论
- 处理流程是否可复现
- 标注是否遮挡关键信号

### 案例 4：机制示意图

**问题**

机制图包含过多箭头和装饰元素，节点命名不一致，读者无法区分假设、证据和推论。

**改进**

- 将证据支持的路径和假设路径用不同线型区分
- 删除与主结论无关的装饰元素
- 统一术语和缩写
- 在图例中解释箭头、抑制线和颜色含义

**检查重点**

- 是否能在 10 秒内看出主机制
- 图中是否混淆已验证结果和推测模型
- 视觉重点是否落在核心结论上

```markdown

核心问题：
这张图要支持的结论：

数据来源：
样本量与重复：
统计方法：

Panel 规划：
- A：
- B：
- C：
- D：

必须标注的信息：
- 坐标轴 / 单位：
- 图例：
- 缩写：
- 误差线：
- 比例尺：

导出要求：
- 格式：
- 尺寸：
- 分辨率：
- 颜色模式：

复现文件：
- 原始数据：
- 分析代码：
- 绘图代码：
- 导出文件：
```

### 图注模板

```markdown
**Figure X. [一句话概括核心发现].**

(A) [Panel A 展示内容、数据来源和关键设置。]
(B) [Panel B 展示内容、比较对象和统计方法。]
(C) [Panel C 展示内容、变量定义和单位。]
(D) [Panel D 展示内容、验证方式或敏感性分析。]

Data are shown as [mean +/- SD / mean +/- SEM / median with IQR / individual data points].
n = [样本量定义，例如 biological replicates / cells / participants].
Statistical test: [检验方法], [是否双侧], [多重比较校正方法].
Abbreviations: [缩写解释].
```

### 文件命名模板

```text
fig01_study-design_v01_2026-05-17.ai
fig02_model-performance_v03_2026-05-17.pdf
fig03_validation_v02_2026-05-17.png
suppfig01_sensitivity-analysis_v01_2026-05-17.pdf
```

推荐规则：

- `figXX` 或 `suppfigXX` 表示图号
- 使用英文短语说明内容
- `v01`, `v02`, `v03` 记录版本
- 使用 `YYYY-MM-DD` 日期格式
- 投稿最终版可另存为 `final`，但仍保留版本历史

## 使用方式

1. 绘图前填写 [图表设计简报](#图表设计简报)
2. 绘图过程中按对应类型清单检查
3. 导出后使用 [投稿前总清单](#投稿前总清单)
4. 返修或协作时维护 [审稿前核对记录](#审稿前核对记录)
